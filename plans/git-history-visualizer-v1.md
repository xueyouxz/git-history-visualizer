## 固定架构决策

- 单个 TypeScript 项目、单个 Node.js 进程；Node.js 提供本地 REST API、SSE 和 React 静态资源。
- 前端使用 React、D3.js、Zustand；D3 只负责图形计算和交互，跨视图状态由 Zustand 保存。
- 不使用 TanStack Query；服务端数据通过类型化 fetch、React hooks 和 AbortController 获取。
- Git 操作只通过参数数组调用系统 Git CLI，不拼接 shell，不执行仓库脚本。
- 基准仓库位于受管根目录，普通 clone 使用 `--no-checkout`，本地源增加 `--no-hardlinks`。
- 只读边界不变：不修改原始仓库，不创建分支，不 commit、push、pull、merge 或 rebase。
- REST 处理仓库、提交、diff 和 worktree；SSE 处理 clone、索引和 fetch 的单向进度。
- 缓存位于仓库外，以仓库身份、完整对象 ID 和分析规则版本定址。
- 典型规模 50～500 个提交，上限约 1,000 个提交；DAG 使用 SVG。
- 默认四区布局为 224px 左栏、中央 DAG、352px 右栏、约 34% 高度底栏；最小桌面宽度 1,200px。
- 视觉系统固定为暖白画布、墨黑正文、低饱和深蓝强调色、有限色盲友好分类色、8px 间距网格、单层分隔线和小圆角；不增加阴影、渐变、发光或主题切换。
- 首版验收平台为 macOS，要求 Node.js 22+、Git 2.40+；Linux 只作为兼容目标。
