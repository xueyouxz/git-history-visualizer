目标文档：`plans/git-history-visualizer-batch-5-11.md`。退出规划模式后，先将本计划原样写入该文件，再启动一个覆盖 #5～#11 的长期目标。

2. #6 贡献者演化

每个 Issue 独立测试、复审、提交、推送和关闭。实现采用贯穿 REST、状态、界面和测试的纵向切片，不并行修改共享工作树。

## 固定架构与公开接口

- 所有 Git 操作继续使用参数数组，禁用 hooks、LFS smudge、外部 diff 和 textconv。除 #9 显式 fetch 与 #10 显式 worktree 操作外，应用保持只读。
- 引入仓库修订指纹：对公开 refs 的名称和完整 OID 排序后哈希。索引、分类和阶段缓存使用“仓库 ID + 修订指纹 + 分析版本”定址，存放在仓库外。
- Zustand 作为唯一跨视图状态源，后续能力复用现有仓库、当前提交、A/B、主线和筛选状态。
- #5 新增 `GET /api/repositories/:id/tree?oid=&path=`，返回目录项的相对路径、类型、对象 ID 和 blob 字节数，不返回 blob 内容，不暴露静态仓库目录。
- #6 新增版本化贡献者演化接口。索引增加不可逆的 `authorId`，由 mailmap 规范化姓名和邮箱生成，接口不返回邮箱。默认窗口为 12 个提交，最多单列 6 名主要贡献者，其余聚合为“其他”。
- #7 新增版本化分类结果接口，并让提交筛选接受多种分类。类型固定为 feature、fix、refactor、test、docs、build/config、merge、mixed。
- #8 新增只读阶段分析接口。用户调整保存在浏览器本地，按仓库修订指纹隔离；原始建议、用户覆盖和原始 DAG 分开保存。
- #9 使用仓库同步任务接口及 SSE：创建、读取、取消和事件流。只执行 fetch、prune 和 tags 更新，成功后原子替换索引。
- #10 使用仓库级 worktree 列表、创建/打开和删除接口。路径固定为受管根目录下 `.worktrees/<repository-id>/<full-oid>`。编辑器配置为绝对可执行文件加参数数组，必须含一个 `{path}` 占位符，禁止 shell 字符串。
- #11 固定 URL v1 状态格式，覆盖仓库、主线、范围、全部筛选、选中提交和 A/B；布局偏好写入 `ghv:layout:v1`。

## 四个执行波次

### 波次一：代码地图与播放，Issue #5

- 以真实 Git tree 数据构建目录嵌套矩形，面积表示 blob 字节数。
- A/B 使用目录并集和稳定排序生成确定性布局；新增、修改、删除和推断重命名同时使用颜色及图案或边框。
- 增加目录下钻、面包屑、文件悬停与相关提交联动。
- 播放模式固定为主线 first-parent 或当前选定拓扑路径。播放只更新 `selectedOid`，支持暂停、取消、重复播放和 reduced-motion。
- 当前底栏先增加 diff 与代码地图标签，为 #6 和 #11 保留分析坞接口。

### 波次二：分析能力闭环，Issue #6 → #7 → #8

- #6：按 DAG 顺序计算固定窗口变更行占比，应用 mailmap，渲染贡献流带和“其他”聚合；选择贡献者联动 DAG 与代码地图，清除后恢复原分析状态。
- #7：实现确定性分类规则、依据、置信度、版本缓存、图例和多类型筛选；不覆盖原始提交数据，不复用 A/B 或选择视觉通道。
- #8：使用 tags、顶层目录变化、配置首次出现、大规模变更和贡献者结构变化评分；禁用真实时间和提交频率，相邻边界至少间隔 8 个提交；支持拖动、关闭、取消、重试和失败隔离。

### 波次三：仓库操作能力，Issue #9 → #10

- #9：构建可追加的真实远程夹具；手动同步更新远程 refs 和 tags，清理失效 refs，增量读取新增对象。同步成功前保留旧索引，成功后保持当前提交、A/B、范围和筛选。
- #10：提取统一配置持久化模块，兼容现有 managed root；只管理工具清单中的 detached worktree。重复打开复用现有目录，路径冲突拒绝覆盖，任何 tracked、untracked 或 ignored 变更都阻止删除。
- 自动测试注入外部启动适配器，不打开真实应用。最终人工验收只申请一次 macOS GUI 权限，验证 Terminal 和配置编辑器。

### 波次四：统一工作台与 V1 验收，Issue #11

- 默认布局固定为 224px 左栏、中央 DAG、352px 右栏和约 34% 高度底栏。
- 底栏模式为代码地图、贡献者流带和并列分析；A/B diff 移入可退出的临时扩大检查器。
- 右栏支持 320～560px，底栏支持折叠和三档高度，退出临时模式恢复原布局。
- 完成 URL 前进、后退、刷新和复制恢复；局部请求失败不得清空其他视图。
- 完成键盘、焦点、文本摘要、非颜色编码、对比度、reduced-motion、1280×800 和 1440×900 验收。
- 新增 macOS 安装、启动、导入、比较、同步和版本工作区说明。
- 通过完整主流程后关闭 #11 和父规格 #1。

## 自动执行与发布规则

- 每个 Issue 开始时读取最新正文和评论，确认依赖已关闭、工作树干净、`main` 与远端同步。
- TDD 接缝预先批准为公开 REST、纯算法结果、Zustand 用户行为和 Playwright 可见流程，不再逐项请求人工确认。
- 每项依次运行针对性测试、全量 Vitest、构建、`git diff --check` 和相关 Playwright。
- 以该 Issue 开始前的提交为固定点执行 Standards 与 Spec 双轴复审，自动修复阻塞发现并复验。
- 验收通过后创建引用 Issue 的独立提交，推送 `main`，评论测试证据并关闭 Issue，再进入下一项。
- 若某项因外部条件阻塞，记录原因并继续不依赖它的 Issue。依赖它的 Issue 自动跳过，批次末集中报告。
- 只有产品含义无法从 #1、当前 Issue 和计划推导，或需要凭据、破坏性操作、GUI 权限时才请求用户输入。

## 测试与完成标准

- Vitest 改为发现全部 `test/**/*.test.ts`；真实 Git 测试继续保持单项 15 秒限制并共享只读夹具。
- 算法测试覆盖稳定矩形布局、重复播放、贡献窗口、分类确定性和阶段评分。
- Git 集成测试覆盖 tree/blob 大小、Unicode、空格路径、重命名、fetch 增量、refs 删除、worktree 冲突及非干净保护。
- Playwright 覆盖代码地图下钻、跨视图联动、播放中断、贡献者选择、分类筛选、阶段失败隔离、同步状态保持、worktree 显式创建和两种目标视口。
- 最终要求：全部 Issue 验收项有自动化证据或明确人工记录，`npm test`、`npm run test:browser`、`npm run build` 和 `git diff --check` 通过，工作树干净且 `main` 与 `origin/main` 同步。
