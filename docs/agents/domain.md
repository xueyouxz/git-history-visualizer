# Domain Docs

本仓库采用 single-context 领域文档结构。

## 探索前读取

- 仓库根目录的 `CONTEXT.md`，若存在。
- `docs/adr/` 中与当前工作相关的架构决策，若存在。

文件尚不存在时直接继续，不主动创建。只有术语或长期决策需要固定时，才通过领域建模流程创建。

## 结构

```text
/
├── CONTEXT.md
├── docs/adr/
└── src/
```

## 使用领域词汇

Issue 标题、规格、测试名称和代码中的领域概念必须采用 `CONTEXT.md` 定义的术语。不要为同一概念随意引入同义词。

若需要的概念尚未定义，先判断它是否属于项目真实领域；确有缺口时记录给领域建模流程处理。

## ADR 冲突

输出若与已有 ADR 冲突，必须明确指出冲突和重新讨论的理由，不得静默覆盖。
